Java 泛型

何为泛型，为什么要使用泛型？

泛型，即为参数化类型，就是将类型由原来的具体的类型参数化，此时类型也定义成参数形式，在调用时传入具体的类型。泛型的使用过程中，操作的数据类型被指定为一个参数，这种参数类型可以用在类、接口和方法中，分别被称为泛型类、泛型接口和泛型方法。

特性

泛型只在编译阶段有效。

结果打印true。说明程序在编译之后会进行泛型擦除。也就是说泛型只在编译阶段有效，在检验泛型类型正确后就会将泛型的相关信息擦除，泛型不会进入运行阶段。因此aClass和bClass都是class java.util.ArrayList。

为了解决这个问题，我们需要用一种方法表示同时是Test<Integer>和Test<Number>父类的引用类型，也就是使用泛型通配符。

类型通配符一般使用?代替具体的类型实参，例如Test<?>代替Test<Number>。

上下界

extends

<? extends XXX>限制了上界，它有副作用，就是不能往里存东西。

test.set(123)不能通过编译。原因是编译器只知道参数是Number的子类，但具体是哪个并不知道。编译器看到后面用123赋值，只是标记一个占位符capture#1，表示捕获了一个Number或Number的子类。后面想set值，编译器不能判断能不能和之前的占位符匹配，所以不被允许。

super

<? super XXX>限制了下界，导致元素没有统一的“根”（除了Object这个大家统一的的根），所以读取元素的时候只能用Object类接收，除非强转。而往里存元素是可行的，限制是只能存XXX的子类，因为这样数据可以自动转型，不然只能强转。

泛型数组

在 Java 中不能创建一个确切的泛型类型的数组。

List<String>[] list = new ArrayList<String>[5];这样的定义是无法通过编译的。而利用通配符创建泛型数组是可行的。

List<?>[] list = new ArrayList<?>[5];
