netty 学习笔记

NIO 基本概念

阻塞、非阻塞

阻塞于非阻塞指的是进程访问数据的时候当数据没有准备就绪时的处理方式。

  • 阻塞：需要等到缓冲区中的数据准备好才去处理其他事情，否则一直等在原地。
  • 非阻塞：无论数据有没有准备好，都直接返回，不会等待。

同步、异步

同步和异步指的是进程和操作系统处理IO事件的方式。

  • 同步：应用程序直接参与IO读写的操作，必须阻塞等待IO事件的完成（阻塞或者轮询）。
  • 异步：把IO事件交给操作系统处理，应用程序等待通知。在等待的期间可以去完成其他的操作。

Java BIO 与 NIO

BIO

BIO是Java传统的IO模型，基于同步阻塞的IO模式，所以所有的交互都是同步并且阻塞的。

NIO

NIO是Java 1.4引入的，基于同步非阻塞的IO模式，提供了Channel、Selector、Buffer等抽象。对于高并发的场景，应该使用NIO模式来开发应用。

Netty

简介

Netty是一个基于NIO的客户端服务器框架，用于快速开发高性能的应用程序。

Netty的主要执行流程：

  • 启动Server，Netty从ParentGroup中选出一个NioEventLoop监听指定的端口。
  • 启动Client，Netty从EventLoopGroup中选出一个NioEventLoop连接Server并处理Server发送的数据。
  • Client连接到Server，创建Channel。
  • Netty从ChildGroup中选出一个NioEventLoop和该Channel绑定，处理Channel中所有的操作。
  • Client通过Channel向Server发送数据。
  • Pipeline中的处理器一次对Channel中的数据包进行处理。
  • Server如果要向Client发送数据，要将数据经过Pipeline的处理器处理封装为ByteBuf数据包进行传输。
  • Server通过Channel将数据包发送给Client。
  • Pipeline中的处理器一次对Channel中的数据包进行处理。

核心组件

Channel

Channel可以看作是数据传输的媒介，服务器和客户端通过Channel进行数据的发送和接收。因此，它可以被打开或关闭、连接或断开连接。

EventLoop 和 EventLoopGroup

EventLoop定义了Netty的核心抽象，用来处理连接的生命周期中发生的事件，在内部，为每一个Channel分配一个EventLoop。而EventLoopGroup是一个EventLoop池，包含很多EventLoop。

Netty给每个Channel分配一个EventLoop，处理用户连接请求、对用户请求进行处理。EventLoop只是一个线程驱动，在其生命周期内只会绑定一个线程，让该线程处理一个Channel的所有IO事件。

一个Channel一旦和一个EventLoop绑定，那么整个生命周期内是不可改变的。一个EventLoop可以和多个Channel绑定。

ServerBootstrap 和 Bootstrap

ServerBootstrap和Bootstrap被称为引导类，指对应用程序进行配置，使其运行起来的过程。Netty处理引导的方式是使应用程序和网络层相互隔离。

Bootstrap是客户端的引导类，在调用bind()（连接UDP）和connect()（连接TCP）时，会创建一个Channel，仅创建一个独立的、没有父Channel的Channel来实现网络交换。

ServerBootstrap是服务端的引导类，在调用bind()时会创建一个ServerChannel接收来自客户端的连接请求，该ServerChannel管理了多个子Channel用于和客户端之间的通信。

ChannelHandler 和 ChannelPipeline

ChannelHandler是对Channel中的数据的处理器，这些处理器可以是系统定义好的也可以是自定义的。这些处理器会被统一添加到一个ChannelPipeline对象中，按照添加的顺序对Channel中的数据依次进行处理。

ChannelFuture

Netty中的IO操作是异步的，所以操作并不会立刻得到返回结果。Netty定义了一个ChannelFuture对象表示异步操作。如果想获得异步操作的返回值，可以通过这个对象的addListener()方法为该异步操作添加监听器，注册回调。

Netty的异步编程模型都是建立在Future与回调概念之上。
