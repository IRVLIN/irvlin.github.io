MyBatis 学习笔记

简介

是什么

MyBatis 是一款持久化框架，一个半 ORM 框架，支持定制化 SQL、存储过程以及高级映射。MyBatis 避免了几乎所有 JDBC 代码和手动设置参数以及获取结果集。MyBatis 可以使用简单的 XML 或者注解来配置和映射原生类型、接口和 Java 的 POJO 为数据库所记录。

ORM

ORM（Object Relational Mapping），对象关系映射，通过使用描述对象和数据库之间映射的元数据，将程序中的对象自动持久化到关系型数据库中。

半自动 ORM 映射

Hibernate 属于全自动 ORM 映射工具，使用 Hibernate 查询关联对象或关联集合对象时，可以根据对象关系模型直接获取，所以是全自动的。MyBatis 则需要手动编写 SQL，因此是半自动的。

传统 JDBC 开发的问题

  • 频繁创建、释放数据库连接对象，很浪费系统资源，影响性能。如果使用连接池可以解决这个问题，但是使用 JDBC 需要自己实现连接池。
  • SQL 语句定义、参数设置、结果集的处理存在硬编码。实际项目中 SQL 语句变化的可能性较大，如果发生变化就要修改代码，系统要重新编译、发布，不易维护。
  • 使用 preparedStatement 向占有位符号传参存在硬编码，SQL 语句的WHERE条件不是一定的，修改 SQL 语句还要修改 Java 代码，因此不易维护。
  • 结果集的处理十分麻烦，如果映射为 Java 对象会比较好处理。

MyBatis如何解决这些问题

  • 使用数据库连接池可以解决频繁创建、释放数据库连接的问题，在 MyBatis 中可以通过mybtais-config.xml配置连接池，很方便。
  • SQL 语句写在代码中不易维护，在 MyBatis 中将 SQL 语句写在XXXMapper.xml 中与 Java 代码分离易于维护。
  • SQL 语句传参麻烦，因为条件不一定，使用 MyBatis 可以自动将 Java 对象映射到 SQL 语句。
  • 结果集的处理很麻烦，使用 MyBatis 可以自动将 SQL 语句执行的结果映射到 Java 对象方便后续的处理。

MyBatis 优点

  • 基于 SQL 语句编程，将 SQL 写在xml文件中，与程序代码解耦，方便维护管理。还支持动态 SQL 语句，很灵活。
  • 不用手动编写开关数据库连接等等冗余的代码，专注于 SQL 的编写。
  • 和 Spring 框架有很好的集成。
  • 支持数据库和对象的映射。
  • 与很多数据库都兼容（只要 JDBC 支持的数据库 MyBatis 都支持）。

MyBatis 缺点

  • SQL 语句需要自己编写，工作量可能比较大。
  • SQL 语句依赖数据库，导致数据库移植性比较差。

Hibernate 和 MyBatis 区别

相同点

都是对 JDBC 的封装，都是持久化框架，都是对 dao 层的开发。

不同点
  • MyBatis 是一个半自动 ORM 框架，配置 Java 对象和 SQL 语句执行结果的对应关系，多表关联关系配置简单；Hibernate 是一个全自动 ORM 框架，配置 Java 对象和数据库表的对应关系，多表关联关系配置复杂。
  • Hibernate 对 SQL 语句封装，提供了日志、缓存、级联等特性，还提供了 HQL 操作数据库，数据库无关性较好；MyBatis 需要手动编写 SQL，支持动态 SQL、存储过程，不支持数据库无关性。

解析和运行原理

编程步骤

  1. 创建SqlSessionFactory
  2. 通过SqlSessionFactory创建SqlSession
  3. 通过SqlSession执行数据库操作
  4. 调用session.commit()提交事务
  5. 调用session.close()关闭会话

工作原理

  1. 读取 MyBatis 配置文件：mybatis.config.xml为全局配置文件，配置了 MyBatis 的运行环境等信息，例如数据库连接信息。
  2. 加载映射文件。该文件配置了操作数据库的 SQL 语句。mybatis-config.xml文件会加载该配置文件，可以加载多个映射文件，每个文件对应数据库中的一张表。
  3. 构造会话工厂：通过 MyBatis 的环境等配置信息构建会话工厂。
  4. 创建会话：通过会话工厂创建会话，包含了所有执行 SQL 语句的方法。
  5. Executor 执行器：MyBatis 维护了一个 Executor 接口来操作数据库，根据会话对象传递的参数动态生成需要执行的 SQL 语句，同时负责查询缓存的维护。
  6. MappedStatement 对象：在 Executor 接口的执行方法中有一个 MappedStatement 类型的参数，该参数是对映射信息的封装，用于存储要映射的 SQL 语句的 id、参数等信息。
  7. 输入参数映射：输入参数类型可以是 Map、List 等集合类型，也可以是基本数据类型和 POJO 类型。输入参数映射过程类似于 JDBC 对 preparedStatement 对象设置参数的过程。
  8. 输出结果映射：输出结果类型可以是 Map、List 等集合类型，也可以是基本数据类型和 POJO 类型。输出结果映射过程类似于 JDBC 对结果集的解析过程。

预编译

定义

SQL 预编译指的是数据库驱动在发送 SQL 语句和参数给 DBMS 之前对 SQL 语句进行编译，这样 DBMS 执行 SQL 时，就不需要重新编译。

为什么要预编译

JDBC 中使用 preparedStatement 来预编译语句，可以优化 SQL 的执行。预编译之后的 SQL 可以直接执行并且重复利用，DBMS 不需要再次编译。预编译还可以防止 SQL 注入，提高了安全性。

Executor 执行器

MyBatis 有三种 Executor 执行器。

  1. SimpleExecutor：每执行一次update或select，就开启一个Statement对象，用完立刻关闭该对象。
  2. ReuseExecutor：执行update或select，以 SQL 作为 key 查找Statement对象，存在就直接使用，否则创建，用完之后不关闭对象，而是放置于Map<String, Statement>中，以供后续重复使用。
  3. BatchExecutor：执行update，将所有 SQL 添加到批处理中，等待统一执行，它缓存了多个statement对象。

在 MyBatis 配置文件中，在设置（settings）可以指定默认的 ExecutorType（执行器类型）。

延迟加载

MyBatis 支持association关联对象和collection关联集合对象的延迟加载。association指的是一对一，collection指的是一对多。在 MyBatis 配置文件中，可以配置是否启用延迟加载。

延迟加载的原理是使用cglib创建目标对象的代理对象，当调用目标方法时，进入拦截器方法，比如调用a.getB().getName()，拦截器invoke()方法发现a.getB()是null值，就会单独发送事先保存好的查询关联 B 对象的 SQL，把 B 查询上来，然后调用a.setB(b)，于是 a 的对象 b 属性就有值了，接着完成a.getB().getName()方法的调用。这就是延迟加载的原理。

映射器

#{} 和 ${} 的区别

  • #{}是占位符，会进行预编译处理；${}是拼接符，没有预编译处理，直接替换字符串。
  • MyBatis 处理#{}时，会将 SQL 中的#{}替换为?，调用preparedStatement的set方法来赋值。
  • MyBatis 处理${}时，直接传入原值，相当于 JDBC 中的statement编译。
  • 变量替换之后，#{}对应的变量自动加上单引号，${}对应的变量不会加单引号。
  • #{}可以防止 SQL 注入，${}不行。
  • #{}的变量替换是在 DBMS 中，${}的变量替换是在 DBMS 外。

模糊查询

  • '%${question}%'会引起 SQL 注入。

  • "%"#{question}"%"。#{}解析成 SQL 语句时，会在变量外侧自动添加单引号，所以%要使用双引号。

  • CONCAT('%', #{question}, '%')。推荐使用。

  • bind标签

获取主键

dao 接口

dao 接口的工作原理是JDK 动态代理，MyBatis 运行的时候会使用 JDK 动态代理为 dao 接口生成代理对象，代理对象会拦截接口方法，转而执行 MappedStatement 代表的 SQL，然后返回结果。

MyBatis 如何封装 SQL 运行结果为目标对象

  • 使用<resultMap>标签，逐一定义列名和对象属性名之间的映射关系。
  • 使用别名，例如SELECT t_name AS name。列名不区分大小写。

有了映射关系后，Mybatis 会利用反射创建对象，给属性逐一赋值然后返回。

A 标签引用（include）了 B 标签的内容，B 标签能否写在 A 后面？

B 标签可以定义在任何地方。MyBatis 按顺序先解析了 xml 文件的 A 标签，A 引用了 B，但 B 还没解析到，所以会被 MyBatis 标记为尚未解析状态，然后继续解析，解析到了 B 标签。解析完成后，Mybatis 会重新解析被标记为尚未解析的标签，这时 B 标签已被解析，A 也可以顺利被解析。

高级查询

MyBatis 是否可以映射枚举类？

MyBatis 可以通过自定义TypeHandler并实现TypeHandler接口的setParameter()和getResult()方法映射任何对象到表的一列上。

TypeHandler可以完成从JavaType至jdbcType的转换，也可以完成jdbcType到JavaType的转换。

动态 SQL

MyBatis 有9种动态 SQL 标签

  • trim
  • where
  • set
  • foreach
  • if
  • choose
  • when
  • otherwise
  • bind

MyBatis 使用OGNL从 SQL 参数对象中计算表达式的值，根据表达式的值动态拼接 SQL。

MyBatis 插件运行原理

MyBatis 仅可以编写针对ParameterHandler、ResultSetHandler、StatementHandler、Executor接口的插件。MyBatis 使用JDK 动态代理，为需要拦截的接口生成代理对象以实现接口方法拦截功能，每当执行这4种接口对象的方法时，就会进入拦截方法，具体是InvocationHandler的invoke()方法，拦截指定配置的需要拦截的方法。

实现 MyBatis 的interceptor接口并且复写intercept()方法，然后给插件写注解，指定拦截哪个接口的哪些方法，最后在配置文件配置编写的插件。

缓存

一级缓存

MyBatis 的一级缓存是基于PerpetualCache的HashMap本地缓存，存储作用域为session。当session执行了flush或者close之后，会将该session的所有cache清空。默认情况下是打开一级缓存的。

二级缓存

二级缓存默认也是采用PerpetualCache，HashMap存储。不同的是存储作用域不同，是Mapper。还可以自定义存储源，例如Redis、Ehcache。默认情况下不开启二级缓存。如果要开启需要实现Serializable序列化接口，在MyBatis配置文件中添加<setting name="cacheEnabled" value="true"，并在配置文件中配置<cache/>。

缓存更新

当缓存相应的作用域执行了增删改之后，该作用域下所有的SELECT中的缓存都会失效。
